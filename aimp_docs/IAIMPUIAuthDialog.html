IAIMPUIAuthDialog
Описание
IAIMPUIAuthDialog упрощает прохождение OAuth2 (и ему подобной) авторизации.

OAuth2-авторизация предполагает, что пользователь выполнит вход в сервис в браузере, а браузер уже передаст авторизационный токен приложению посредством url-обратного вызова. На время отображения диалога, AIMP поднимает локальный сервер по адресу localhost:8090 и ждет уведомления от сервиса, которое будет передано плагину посредством IAIMPUIAuthDialogEvents.

Пользователь может отменить процесс авторизации, просто закрыв диалоговое окно. В этом случае, функция Execute завершится с кодом E_FAIL.
Методы
Execute
inOwnerWndHWND Дескриптор окна-владельца диалога.
Если указать 0 - будет использоваться дескриптор главного окна.
inTitleIAIMPString Название сервиса, для которого вызван диалог. Показывается пользователю в диалоге.
inTextIAIMPString Пояснение по авторизации.
Передайте nil/null, чтобы использовать пояснение по умолчанию.
inEventsHandlerIAIMPUIAuthDialogEvents
inFlagsLongWord 0, для будущего использования.
outResultHRESULTS_OK - Метод успешно завершился
E_INVALIDARG - Один из параметров задан неверно
E_NOTIMPL - Метод не реализован
E_UNEXPECTED - Произошел непредвиденный сбой
E_ACCESSDENIED - Метод был вызван не из главного потока
Доступ к интерфейсу
<IAIMPServiceUI>.QueryInterface(IID_IAIMPUIAuthDialog, <IAIMPUIAuthDialog>)
