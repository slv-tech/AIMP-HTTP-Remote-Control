IAIMPMLDataProvider2
Описание
Интерфейс является основополагающим - он предоставляет доступ к данным хранилища. Должен быть реализован на том же уровне, что и IAIMPMLExtensionDataStorage.

Вы должны использовать эту версию интерфейса вместо IAIMPMLDataProvider, если ваше хранилище использует постраничный доступ к данным.

Реализация интерфейса должна поддерживать асинхронную работу.
Методы
GetData
inFieldsIAIMPObjectList Список имен полей, для которых запрашиваются данные.
Список строится автоматически, в зависимости от того, какие поля видны пользователю, являются обязательными или связующими.
inFilterIAIMPMLDataFilter Настройки фильтрации данных.
Плагин обязан учитывать его, если флаг AIMPML_DATASTORAGE_CAP_FILTERING присутствует в AIMPML_DATASTORAGE_PROPID_CAPABILITIES.
inReservedIUnknown Параметр зарезервирован для будущего использования. На текущий момент всегда равен nil / null.
varPageIDIUnknown Параметр используется для постраничной навигации по хранилищу данных, если оно поддерживает такой режим.

Входное значение:
ID страницы, с которой нужно загрузить данные. Значение nil / null указывает на то, что это первая страница.

Выходное значение:
ID следующей страницы, если она есть, или nil / null, если ее нет.
outDataIUnknown Ответ на запрос. Может принимать следующие значения в случае успешного выполнения функции:
  • IAIMPMLDataProviderSelection - выборка данных
  • IAIMPString - информационный многострочный текст (с поддержкой BB-форматирования), выводится пользователю в таблице фонотеки.
outResultHRESULTS_OK - в случае успеха, в случае ошибки - любое другое значение.
