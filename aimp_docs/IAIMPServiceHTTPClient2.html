IAIMPServiceHTTPClient2
Описание
Этот сервис представляет собой более общую версию IAIMPServiceHTTPClient сервиса. Позволяет плагину отправлять HTTP-запросы к удаленным серверам.

Все запросы выполняют в отдельном потоке - плагину не нужно создавать свой поток.
Как только запрос будет выполнен - плагин получит об уведомление через обработчик событий (см.IAIMPHTTPClientEvents)
Методы
Request
inURLIAIMPStringДля создания защищенного подключения, необходимо указать в URL "https://"

Вы можете указать свои поля с данным для HTTP-заголовка, для этого их необходимо передать в строке с URL, отделив каждое поле символами переноса строки (CRLF).

Например:
'https://example.org/'#13#10'Заголовок 1: Данные'#13#10'Заголовок 2: Данные'
inMethodLongWord Метод запроса, может быть одним из следующих значений:
  • AIMP_SERVICE_HTTPCLIENT_METHOD_GET
  • AIMP_SERVICE_HTTPCLIENT_METHOD_POST
  • AIMP_SERVICE_HTTPCLIENT_METHOD_PUT
  • AIMP_SERVICE_HTTPCLIENT_METHOD_DELETE
  • AIMP_SERVICE_HTTPCLIENT_METHOD_HEAD
    • inFlagsLongWord0 или комбинация из флагов:
  • AIMP_SERVICE_HTTPCLIENT_FLAGS_WAITFOR - этот метод не вернет управление до тех пор, пока операция не будет завершена. Не рекомендуется
  • AIMP_SERVICE_HTTPCLIENT_FLAGS_UTF8 - в этом случае строка, содержащая URL, будет преобразована в UTF8 кодировку
  • AIMP_SERVICE_HTTPCLIENT_FLAGS_PRIORITY_XXX - приоритет выполнения задачи, см. IAIMPTaskPriority
    • inAnswerDataIAIMPStream Экземпляр бинарного поток, куда будет записан ответа от сервера.
    inPostDataIAIMPStream Данные, которые необходимо отправить серверу.
    Может быть равен nil/null.

    Только для Method = AIMP_SERVICE_HTTPCLIENT_METHOD_POST / AIMP_SERVICE_HTTPCLIENT_METHOD_PUT.
    inEventsHandlerIAIMPHTTPClientEvents см. так же IAIMPHTTPClientEvents2
    inParamsIAIMPConfig Для будущего использования, должен быть равен nil / null

    • outResultHRESULTS_OK - Метод успешно завершился
