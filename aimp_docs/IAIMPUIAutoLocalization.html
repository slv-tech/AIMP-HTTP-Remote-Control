Автоматическая локализация
Общие сведения
Движок программы автоматически локализирует все создаваемые им окна и элементы управления. Для того, чтобы созданные плагином окна, так же автоматически локализовались, необходимо правильным образом подготовить файл локализации, и не забыть указать всем элементам управления имена.
Правила
1. Имя секции в файле локализации должно соответствовать имени IAIMPUIForm.


2. Имя ключа секции должно соответствовать имени IAIMPUIControl, в этом случае значение ключа будет применено к заголовку элементу управления.


3. Для локализации текстовой подсказки элемента управления, нужно создать ключ с именем этого элемента управления + ".h"

[Имя формы]
MyButton.h=Подсказка для кнопки
4. Для локализации списка Combobox-элементов используется отдельная секция, оформленная по следующему правилу: 

[Имя формы.Имя элемента]
i[0]=Запись 1
i[1]=Запись 2
5. Для локализации встроенных кнопок наследника IAIMPUIBaseButtonnedEdit используется отдельная секция, оформленная по следующему правилу: 

[Имя формы.Имя элемента]
b[0]=Кнопка 1
b[1]=Кнопка 2
6. Аналогичное правило применяется для локализации колонок IAIMPUITreeList-а: 

[Имя формы.Имя элемента]
c[0]=Колонка 1
c[1]=Колонка 2
7. Для локализации TextHint-а IAIMPUIEdit используется отдельная секция, содержащая ключ "th": 

[Имя формы.Имя элемента]
th=Подсказка для поля ввода
8. Для автоматической локализации пунктов меню используется отдельная, глобальная секция. Вместо имени элемента используется ID: 

[Common]
<Action.ID>=Запись 1
<Action.ID>.h=Подсказка для записи 1
<Action.ID>.g=Имя группы записи 1
<MenuItem.ID>=Запись 2
