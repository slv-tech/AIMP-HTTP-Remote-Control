IAIMPPlaylist
Описание
Экземпляр интерфейса IAIMPPlaylist предоставляет доступ к конкретному плейлисту.
Свойства

Методы
Add
inObjectIUnknownIAIMPString с FileURI или IAIMPFileInfo (при наличии флага AIMP_PLAYLIST_ADD_FLAGS_FILEINFO);
inFlagsLongWord 0 или комбинация из флагов:
  • AIMP_PLAYLIST_ADD_FLAGS_NOCHECKFORMAT - флаг подавляет проверку на поддерживаемые типы при добавлении файлов в плейлист
  • AIMP_PLAYLIST_ADD_FLAGS_NOEXPAND - флаг позволяет добавить в плейлист файлы из списка как есть, не раскрывая их на виртуальные файлы
  • AIMP_PLAYLIST_ADD_FLAGS_NOASYNC - добавление файлов в плейлист будет происходить в главном потоке, управление не будет передано плагину до тех пор, пока операция не будет завершена (не рекомендуется)
  • AIMP_PLAYLIST_ADD_FLAGS_FILEINFO - флаг показывает, что в качестве объекта передан IAIMPFileInfo (в противном случае объект трактуется как IAIMPString)
    • inInsertInInteger = -2 - добавление записей в случайную позицию плейлиста
    = -1 - добавление записей в конец плейлиста
    >= 0 - добавление записей в указанную позицию

    outResultHRESULTS_OK - Метод успешно завершился
    E_ACCESSDENIED - Метод был вызван не из главного потока
    E_HANDLE - Объект, к которому привязан интерфейс, был убит
    E_INVALIDARG - Один из параметров задан неверно
    E_NOTIMPL - Метод не реализован
    E_UNEXPECTED - Произошел непредвиденный сбой

    Close
    Метод закрывает плейлист. В зависимости от настроек, плейлист будет либо числиться среди выгруженных, либо будет удален.
    inFlagsLongWord Может принимать 0 или одно из значений:
    • AIMP_PLAYLIST_CLOSE_FLAGS_FORCE_REMOVE - Удалить плейлист после закрытия, несмотря на текущие настройки
    • AIMP_PLAYLIST_CLOSE_FLAGS_FORCE_UNLOAD - Выгрузить плейлист после закрытия, несмотря на текущие настройки

    По умолчанию - 0.
