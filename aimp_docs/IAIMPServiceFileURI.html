IAIMPServiceFileURI
Описание
Сервис IAIMPServiceFileURI представляет методы для работы с полными именами реальных и виртуальных файлов.
Методы
Build
Метод строит FileURI на основе имени файла-контейнера и имени трека. См. соглашение имен виртуальных файлов.
inContainerFileNameIAIMPString
inPartNameIAIMPString
outFileURIIAIMPString
outResultHRESULTS_OK - Метод успешно завершился
E_INVALIDARG - Один из параметров задан неверно
E_NOTIMPL - Метод не реализован
E_UNEXPECTED - Произошел непредвиденный сбой
Parse
Метод разбирает FileURI, и возвращает имя файла-контейнера и имя трека.
Если FileURI ссылается на реальный файл, то PartName будет установлена в nil/null
inFileURIIAIMPString
outContainerFileNameIAIMPString
outPartNameIAIMPString
outResultHRESULTS_OK - Метод успешно завершился
E_INVALIDARG - Один из параметров задан неверно
E_NOTIMPL - Метод не реализован
E_UNEXPECTED - Произошел непредвиденный сбой
ChangeFileExt
Метод позволяет изменить расширение файла.
varFileURIIAIMPString
inNewExtIAIMPString Новое расширение в формате: .mp3
inFlagsLongWord 0 или комбинация из флагов:

AIMP_SERVICE_FILEURI_FLAG_DOUBLE_EXTS - двойное расширение будет заменено на то, что указано в NewExt. Если же флаг не задан - заменится лишь последнее расширение.

AIMP_SERVICE_FILEURI_FLAG_PART_EXT - при наличии флага будет обрабатываться расширение у трека, в противном случае - у файла-контейнера.
outResultHRESULTS_OK - Метод успешно завершился
E_INVALIDARG - Один из параметров задан неверно
E_NOTIMPL - Метод не реализован
E_UNEXPECTED - Произошел непредвиденный сбой
ExtractFileExt
Метод позволяет узнать расширение файла.
inFileURIIAIMPString
outExtIAIMPString Расширение в формате: .mp3
inFlagsLongWord 0 или комбинация из флагов:

AIMP_SERVICE_FILEURI_FLAG_DOUBLE_EXTS - флаг включает поддержку двойных расширений у файлов.

AIMP_SERVICE_FILEURI_FLAG_PART_EXT - при наличии флага будет браться расширение у трека, в противном случае - у файла-контейнера.
outResultHRESULTS_OK - Метод успешно завершился
E_INVALIDARG - Один из параметров задан неверно
E_NOTIMPL - Метод не реализован
E_UNEXPECTED - Произошел непредвиденный сбой
ExtractFileName
Метод позволяет получить имя файла без пути.
В имя файла будет включено как имя файла-контейнера, так и имя трека.
inFileURIIAIMPString
outSIAIMPString
outResultHRESULTS_OK - Метод успешно завершился
E_INVALIDARG - Один из параметров задан неверно
E_NOTIMPL - Метод не реализован
E_UNEXPECTED - Произошел непредвиденный сбой

Метод позволяет получить имя родительской папки реального файла или файла-контейнера.
inFileURIIAIMPString

ExtractFileParentName
Метод позволяет получить имя родителя:
Если FileURI ссылается на виртуальный файл - функция вернет имя файла-контейнера
Если FileURI ссылается на реальный файла - функция вернет имя родительской папки
