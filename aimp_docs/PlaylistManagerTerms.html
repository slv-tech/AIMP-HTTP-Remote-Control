Основные сведения
Общие сведения
AIMP позволяет работать с одним или несколькими плейлистами, при этом плейлисты делятся на загруженные - отображаются в программе, и незагруженные - присутствуют в менеджере плейлистов, но не загружены в память и не отображаются среди вкладок плейлистов.
Хранение
Все плейлисты, независимо от того, как они были созданы - открыты из внешних источников, созданы в программе - хранятся в папке профиля в формате *.aimppl.

Таким образом, если плейлист загружается с диска - создается локальная копия в папке профиля и вся дальнейшая работа идет с ней.
Автоматическое имя
Если у плейлиста не определено имя - оно рассчитывается на основе его содержимого - берется имя родительской для всех файлов папки.
Настройки отображения
Помимо глобальных для всех плейлистов настроек, АИМП позволяет модифицировать настройки отображения на уровне каждого из плейлистов. Плагин имеет доступ к настройкам уровня плейлиста.
Курсоры
Каждый плейлист хранит дополнительную информацию о позиции воспроизведения - номер проигрываемого трека (PlayingIndex), и курсор воспроизведения (PlaybackCursor).

PlaybackCursor - курсор воспроизведения плейлиста, он не учитывает пользовательскую очередь и плагины. Значение курсора указывает на значение свойства PlaybackQueueIndex у проигрываемой в данный момент записи, и, в общем случае, не совпадает с порядком номером записи в плейлисте.

PlayingIndex просто указывает на текущий проигрываемый файл в этом плейлисте. Обратите внимание, что у каждого плейлиста свой PlayingIndex, корректное значение которого не говорит о том, что сам плейлист в данный момент проигрывается - это можно узнать с помощью IAIMPServicePlaylistManager.GetPlayingPlaylist.
