IAIMPExtensionAudioDecoderOld
Описание
IAIMPExtensionAudioDecoderOld, в отличие от IAIMPExtensionAudioDecoder, создает декодер на основе реального или виртуального имени файла, а не IAIMPStream.

Интерфейс де-факто устаревший, используется для совместимости со старыми декодерами, что не поддерживают работу с пользовательскими потоками данных. Плагин должен стараться избегать его использования, т.к. в этом случае пользователь лишается следующего функционала: возможности кэширования исходного файла, проигрывания аудиофайлов из архивов, проигрывания отдельных треков из сета.

Реализация интерфейса должна поддерживать асинхронную работу.
Методы
CreateDecoder
inFileNameIAIMPString Полный путь к файлу на диске. Путь может быть виртуальным.
inFlagsLongWord0 или комбинация из следующих флагов:
  • AIMP_DECODER_FLAGS_FORCE_CREATE_INSTANCE - Флаг предназначен только для тех декодеров, что не имеют возможности работать с несколькими экземплярами. Этот флаг указывает, что все активные в данный момент декодеры должны быть аннулированы, для возможности создания нового экземпляра декодера
    • inErrorInfoIAIMPErrorInfoИнформация об ошибке, заполняется плагином в случае ее возникновения.

    Параметр может быть равен nil (null).

    Внимание! Плагин не должен заполнять информацию об ошибке, если он не смог распознать в потоке/файле поддерживаемый им формат.
    outDecoderIAIMPAudioDecoder Экземпляр декодера
    outResultHRESULTS_OK - в случае успеха, в случае ошибки - любое другое значение.
    E_PENDING - Декодер не может быть создан, поскольку уже есть активные экземпляры (только в том случе, если декодер не поддерживает работу с несколькими экземплярами)
    Создание экземпляра
    Экземпляр создается плагином и регистрируется в программе при инициализации плагина при помощи IAIMPCore.RegisterExtension.

    Вместе с данным интерфейсом плагин должен зарегистрировать IAIMPExtensionFileFormat, которое позволит программе узнать о новых поддерживаемых форматах.
