IAIMPMLExtensionDataStorage
Описание
Расширение для IAIMPServiceMusicLibrary, позволяет плагину реализовать свое собственное хранилища данных для фонотеки. На уровне этого интерфейса должны быть реализованы IAIMPMLDataProvider и какие-либо команды.

Реализация интерфейса должна поддерживать асинхронную работу.
Предок
IAIMPMLExtensionDataStorage наследуется от IAIMPPropertyList
Свойства
Доступные идентификаторы свойств для методов интерфейса IAIMPPropertyList:

AIMPML_DATASTORAGE_PROPID_ID
Уникальный идентификатор хранилища данных
ТипIAIMPString
ЧтениеIAIMPPropertyList.GetValueAsObject(.., IID_IAIMPString, ..)
AIMPML_DATASTORAGE_PROPID_CAPTION
Название хранилища данных. Отображается пользователю.
ТипIAIMPString
ЧтениеIAIMPPropertyList.GetValueAsObject(.., IID_IAIMPString, ..)
AIMPML_DATASTORAGE_PROPID_CAPABILITIES
Список поддерживаемых опций. 0 или комбинация флагов:
  • AIMPML_DATASTORAGE_CAP_FILTERING - хранилище поддерживает фильтрацию данных на своей стороне. В противном случае - вся фильтрация будет выполняться на уровне движка фонотеки
  • AIMPML_DATASTORAGE_CAP_GROUPINGPRESETS - разрешает пользователю создавать свои шаблоны группировки данных.
  • AIMPML_DATASTORAGE_CAP_PREIMAGES - разрешает пользователю создать прообразы на основе этого хранилища данных (не распространяется на IAIMPExtensionPlaylistPreimageFactory).

  • AIMPML_DATASTORAGE_CAP_CUSTOMIZEGROUPS - разрешает пользователю настраивать список полей, что используются для группировки данных в таблице и для отображения информации о группе.
  • AIMPML_DATASTORAGE_CAP_NOBOOKMARKS - отключает сохранение / восстановление выбранного пользователем узла в дереве группировки. Может быть полезным, если восстановление занимает много времени.

ТипInteger
ЧтениеIAIMPPropertyList.GetValueAsInt32
Методы
Initialize
Метод вызывается после успешной регистрации расширения.
inManagerIAIMPMLDataStorageManager Через этот интерфейс плагин будет сообщать фонотеке об изменениях в своем хранилище данных.
Finalize
Метод вызывается при завершении работы с хранилищем данных при закрытии программы. Плагин должен "отпустить" все внешние ссылки.
FlushCache
Метод вызывается, когда пользователь нажимает клавишу F5 или кнопку "обновить". После этого вызова движок программы перезапрашивает данные из хранилища.
inReservedInteger Для будущего использования, должно быть равно 0
ConfigLoad
Внимание: метод вызывается при первой активации вкладки хранилища в фонотеке
inConfigIAIMPConfig
inSectionIAIMPString Имя секции, в которую должны быть сохранены настройки.
Допускается создание дочерних секций: Section + "." + ChildSectionName
outResultHRESULTS_OK - в случае успеха, в случае ошибки - любое другое значение.
ConfigSave
inConfigIAIMPConfig
inSectionIAIMPString Имя секции, в которую должны быть сохранены настройки.
Допускается создание дочерних секций: Section + "." + ChildSectionName
outResultHRESULTS_OK - в случае успеха, в случае ошибки - любое другое значение.
GetFields
Метод должен вернуть настройки и расположение полей с данными в зависимости от указанной Schema
inSchemaInteger Одно из следующих значений:
  • AIMPML_FIELDS_SCHEMA_ALL - список всех возможных полей в хранилище данных. List должен содержать IAIMPMLDataField
  • AIMPML_FIELDS_SCHEMA_TABLE_GROUPBY - список полей, участвующих в группировке данных в таблице. Если список пустой - режим отображения в виде "карточек" будет недоступен пользователю. List должен содержать имена полей в виде IAIMPString
  • AIMPML_FIELDS_SCHEMA_TABLE_GROUPDETAILS - список полей для таблицы, данные из которых отображаются в зоне карточек слева от списка. List должен содержать имена полей в виде IAIMPString
  • AIMPML_FIELDS_SCHEMA_TABLE_VIEW_DEFAULT - список полей видимых по умолчанию колонок в таблице в режиме отображения "таблица". List должен содержать имена полей в виде IAIMPString
  • AIMPML_FIELDS_SCHEMA_TABLE_VIEW_GROUPDETAILS - список полей видимых по умолчанию колонок в таблице в режиме отображения "карточки". List должен содержать имена полей в виде IAIMPString
  • AIMPML_FIELDS_SCHEMA_TABLE_VIEW_ALBUMTHUMBNAILS - список полей видимых по умолчанию колонок в таблице в режиме отображения "миниатюры". List должен содержать имена полей в виде IAIMPString

Внимание!
Хранилище данных обязано содержать поля со следующими именами:
  • AIMPML_RESERVED_FIELD_ID - уникальный идентификатор записи в БД
  • AIMPML_RESERVED_FIELD_FILENAME - имя файла, именно его плеер использует для проигрывания и прочих операций
Без этих полей невозможна регистрация расширения в фонотеке.


Помимо этого, движок фонотеки предусматривает наличие (опционально) следующих полей:
  • AIMPML_RESERVED_FIELD_FILESIZE - размер файла в байтах (тип поля: Int64), используется при расчетах объема содержимого в таблице
  • AIMPML_RESERVED_FIELD_DURATION - длительность в секундах (тип поля: Double), используется при расчетах объема содержимого в таблице
  • AIMPML_RESERVED_FIELD_USERMARK - пользовательская оценка (тип поля: Double), см. IAIMPMLDataStorageCommandUserMark
Для каждого поля создается колонка с преднастроенным набором свойств.
outListIAIMPObjectList Список должен содержать либо IAIMPMLDataField, либо IAIMPString - в зависимости от Schema.
outResultHRESULTS_OK - в случае успеха, в случае ошибки - любое другое значение.
GetGroupingPresets
Плагин должен заполнить список Presets предопределенными шаблонами для дерева группировки, в зависимости от Schema.
inSchemaInteger Одно из следующих значений:
  • AIMPML_GROUPINGPRESETS_SCHEMA_BUILTIN - "встроенные" шаблоны, пользователь не может их ни удалить, ни редактировать
  • AIMPML_GROUPINGPRESETS_SCHEMA_DEFAULT - предопределенные пользовательские шаблоны (могут быть удалены или отредактированы пользователем)

Внимание!
Плагин должен добавить по крайней мере один BuildIn-шаблон.
inPresetsIAIMPMLGroupingPresets
outResultHRESULTS_OK - в случае успеха, в случае ошибки - любое другое значение.
Создание экземпляра
Экземпляр создается плагином и регистрируется в программе при инициализации плагина при помощи IAIMPCore.RegisterExtension с указанием соответствующего идентификатора сервиса.
