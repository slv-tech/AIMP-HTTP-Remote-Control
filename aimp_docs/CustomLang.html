Пользовательские локализации
Описание
Если плагин имеет собственные файлы локализации - он должен разместить их в подпапке Langs своей рабочей папки в формате *.lng. Формат LNG представляет собой обычный INI-файл.

Локализации автоматически подхватываются движком плеера и добавляются в общий стэк, доступ к которому предоставляет сервис IAIMPServiceMUI.

Автоматический подхват осуществляется по следующим правилам:
1. При загрузке локализации/плагина движок ищет в папке <PluginWorkPath>\Langs файл с таким же именем и расширением, как у основной локализации.
2. Если файла с таким именем нет - берется English.lng
3. Если файла с таким именем нет - берется Russian.lng
4. Содержимое файла сливается с основной локализацией в памяти (файл основной локализации не меняется) без замены одинаковых параметров.
