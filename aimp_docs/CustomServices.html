Создание собственных сервисов
Требования к интерфейсам
1. Каждый интерфейс независимо от типа (сервис, примитив и т.п.) должен иметь уникальный GUID
2. GUID не должен совпадать с GUID-ами стандартных интерфейсов
3. Категорически запрещено менять состав любого интерфейса без смены его GUID-а, если он хотя бы раз публиковался
Регистрация расширений
1. Сервис обязан реализовать интерфейс IAIMPServiceAttrExtendable*, если он поддерживает расширения
2. Сервис не должен реализовывать свои методы для регистрации расширений
3. Регистрация расширений должна происходить строго через IAIMPCore.RegisterExtension - это дает возможность ядру программы отслеживать связи между плагинами и автоматически отключать расширения при отгрузке плагина.
Регистрация примитивов
Сервис обязан реализовать интерфейс IAIMPServiceAttrObjects*, если он имеет собственные примитивы и хочет предоставить к ним доступ другим плагинам. При этом другие плагины смогут получить доступ к примитивам через IAIMPCore.CreateObject.
Регистрация сервиса
Регистрация сервиса выполняется с помощью метода IAIMPCore.RegisterService при инициализации программы. Отключение сервиса происходит автоматически перед финализацией плагина.
