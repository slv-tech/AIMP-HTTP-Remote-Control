IAIMPServicePlayer
Описание
Сервис предоставляет доступ к объекту управления воспроизведением.
Свойства
Доступные идентификаторы свойств для методов интерфейса IAIMPPropertyList:
Доступ к IAIMPPropertyList осуществляется через <IAIMPServicePlayer>.QueryInterface(IID_IAIMPPropertyList, ..);
AIMP_PLAYER_PROPID_ACTION_ON_END_OF_TRACK
0 - действие по умолчанию
1 - перейти на следующий трек и остановиться
2 - перейти на следующий трек и встать на паузу

Play4
inFileURIIAIMPStringFileURI, может содержать путь к плейлисту
inFlagsLongWord 0 или комбинация из флагов:
  • AIMP_SERVICE_PLAYER_FLAGS_PLAY_FROM_PLAYLIST - плеер проводит поиск файла среди загруженных плейлистов, в случае успеха - проигрывает его.
  • AIMP_SERVICE_PLAYER_FLAGS_PLAY_FROM_PLAYLIST_CAN_ADD - флаг используется совместно с предыдущим, если файл не найден - он добавляется в плейлист (плейлист выбирается согласно настройкам открытия файлов из внешних приложений), затем начинается его воспроизведение
  • AIMP_SERVICE_PLAYER_FLAGS_PLAY_WITHOUT_ADDING_TO_PLAYLIST - флаг указывает, что плеер может проиграть файл, не добавляя его в плейлист
  • AIMP_SERVICE_PLAYER_FLAGS_PLAY_SUSPENDED - благодаря этому флагу, плеер запустит файл и встанет на паузу
Примечание 1: Обратите внимание, что функция не учитывает, началось ли реально воспроизведение файла или нет, она лишь инициирует загрузку файла в отдельном потоке (см. AIMP_MSG_EVENT_STREAM_START)

Примечание 2: Функция вернет E_FAIL, если файл не удовлетворяет условиям, заданным в виде флагов.

Примечание 3: В случае, если FileURI содержит путь к плейлисту, флаг AIMP_SERVICE_PLAYER_FLAGS_PLAY_WITHOUT_ADDING_TO_PLAYLIST игнорируется, т.к. проигрывание плейлиста без его импорта невозможно.
