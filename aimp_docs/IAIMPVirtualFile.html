IAIMPVirtualFile
Описание
Интерфейс IAIMPVirtualFile предоставляет информацию о виртуальном файле и методы для работы с ним.

Реализация интерфейса должна поддерживать асинхронную работу.
Предок
IAIMPVirtualFile наследуется от IAIMPPropertyList.
Свойства
Доступные идентификаторы свойств для методов интерфейса IAIMPPropertyList.

AIMP_VIRTUALFILE_PROPID_INDEXINSET
Свойство указывает на порядковый номер виртуального файла в сете.
ТипInteger
ЧтениеIAIMPPropertyList.GetValueAsInt32
ЗаписьIAIMPPropertyList.SetValueAsInt32
AIMP_VIRTUALFILE_PROPID_CLIPSTART / AIMP_VIRTUALFILE_PROPID_CLIPFINISH
Свойства позволяют указать начало и конец отрывка для проигрывания (в секундах).
Если виртуальный файл не имеет ограничений по времени - метод IAIMPPropertyList.GetValueAsFloat должен вернуть для этих свойств E_NOTIMPL.
ТипDouble
ЧтениеIAIMPPropertyList.GetValueAsFloat
ЗаписьIAIMPPropertyList.SetValueAsFloat
AIMP_VIRTUALFILE_PROPID_AUDIOSOURCEFILE
Реальное имя аудиофайла, на который ссылается виртуальный файл.
Свойство может отсутствовать в том случае, если реального аудио файла нет, или имя аудиофайла совпадает с именем файла-контейнера. В случае отсутствия свойства метод IAIMPPropertyList.GetValueAsObject должен возвращать для этого свойства E_NOTIMPL.

Пример 1:
    Файл разметки представлен в виде отдельного файла - C:\Music\Track.cue. Имя виртуального файла, соответственно, будет выглядеть как C:\Music\Track.cue:1. В таком случае AIMP_VIRTUALFILE_PROPID_AUDIOSOURCEFILE будет возвращать имя аудиофайла, для которого автоматикой был подхвачен файл разметки, например: C:\Music\Track.mp3.

Пример 2:
    Допустим, что файл разметки расположен в тегах аудиофайла. Имя виртуального файла, в таком случае, будет выглядеть как C:\Music\Track.mp3:1. В данной ситуации AIMP_VIRTUALFILE_PROPID_AUDIOSOURCEFILE не нужен - метод GetValueAsObject вернет E_NOTIMPL.

ТипIAIMPString
ЧтениеIAIMPPropertyList.GetValueAsObject(ID, IID_IAIMPString, ..)
ЗаписьIAIMPPropertyList.SetValueAsObject(ID, <IAIMPString>)
AIMP_VIRTUALFILE_PROPID_FILEFORMAT
Формат файла в сокращенном виде. Например: MP3, OGG, CDA.

Свойство необязательное, в случае его отсутствия движок программы вычислит формат файла по расширению файла-контейнера.
ТипIAIMPString
ЧтениеIAIMPPropertyList.GetValueAsObject(.., IID_IAIMPString, ..)
AIMP_VIRTUALFILE_PROPID_FILEURI
имя виртуального файла
ТипIAIMPString
ЧтениеIAIMPPropertyList.GetValueAsObject(ID, IID_IAIMPString, ..)
ЗаписьIAIMPPropertyList.SetValueAsObject(ID, <IAIMPString>)
Методы
CreateStream
Метод создает экземпляр IAIMPStream.

Если виртуальный файл ограничивает реальный лишь по времени проигрывания (см. AIMP_VIRTUALFILE_PROPID_FLOAT_CLIPXXX) - он может не создавать поток сам, а оставить это на откуп ядра программы. Для этого метод должен вернуть код E_NOTIMPL.
outStreamIAIMPStream
outResultHRESULTS_OK - Метод успешно завершился
E_INVALIDARG - Один из параметров задан неверно
E_NOTIMPL - Метод не реализован
E_UNEXPECTED - Произошел непредвиденный сбой
GetFileInfo
Метод возвращает расширенную информацию о файле
inInfoIAIMPFileInfo
outResultHRESULTS_OK - в случае успеха, в случае ошибки - любое другое значение.
IsExists
Метод проверяет наличие источника
outResultLongBool
IsInSameStream
Метод показывает будут ли созданные через CreateStream потоки текущего и передаваемого в VirtualFile экземпляров интерфейса ссылаться на одну и ту же область данных.

Если информация по каким-либо причинам не может быть рассчитана - необходимо вернуть E_FAIL.

Метод служит лишь для оптимизации при последовательном проигрывании сетов.
inVirtualFileIAIMPVirtualFile
outResultHRESULTS_OK - в случае успеха, в случае ошибки - любое другое значение.
Synchronize
При вызове этого метода экземпляр интерфейса должен проверить верность своих внутренних данных, например: присутствует ли реальный файл, на часть которого он ссылается, не "съехала" ли эта часть внутри файла и т.п.. В случае возврата ошибки, ядро автоматически перезагрузит файл-контейнер с помощью IAIMPExtensionFileExpander.

Метод вызывается ядром программы перед вызовом CreateStream.
outResultHRESULTS_OK - в случае успеха, в случае ошибки - любое другое значение.
Создание экземпляра
Экземпляр создается плагином и добавляется в список при вызове метода IAIMPExtensionFileExpander.Expand.
